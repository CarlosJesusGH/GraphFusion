    File Formats

    The preferred format for network files in GraphFusion is Edge List or variations of it depending on the type of network. The Edge List format is a simple text file with one edge per line. Each line contains two node names separated by a tab or a space. The node names can be any string of characters, but they cannot contain tabs or spaces. The node names are case sensitive.

    Adjacency matrices are also allowed, but they are not recommended for large networks. The adjacency matrix is a square matrix with the same number of rows and columns as the number of nodes in the network. The value of each element in the matrix represents the weight of the edge between the corresponding row and column nodes. The adjacency matrix can be saved in a CSV file.

    For more information please refer to the example netwoks in the 'example_networks' directory in this repository, or check them out online on the Github repository.

    Following there is a list of the supported network types and one example for each.

    Undirected networks represent bidirectional relationships between entities without a preferred directionality. They are used to model systems where interactions between entities are symmetric. For example, a PPI network where the relationship between two proteins is the same regardless of which protein is the source and which is the target.

    They are networks where hyperedges rather than traditional edges can connect nodes. Hyperedges can connect multiple nodes simultaneously, representing complex relationships between entities, for example, a gene regulatory network where a single transcription factor can regulate a group of genes. To analyze these networks, we use hyper graphlets that extend the concept of graphlets to hyper networks.

    For the input file, each line represents a hyperedge. The first element in the line is the name or id of the hyperedge, and the rest of the elements are the names of the nodes connected by the hyperedge.
